Tailwind NG provides a wide range of predesigned components to help you get started quickly. However, you may need to customize a component to match your design system or brand guidelines.

In this guide, you will learn the various ways you can customize the Tailwind NG Badge to create a unique design for your application and make it stand out. These approaches are listed from specific to generic. Depending on your need, choose the best approach or combine them to get the best results without repeating yourself.

This guide assumes you have already installed Tailwind NG . If not, please install it before continuing.

01. Use class attribute

Use the class attribute when you need to apply very specific style to the component.

02. Use classList

Use the (componentRef).classList.value signal to apply very specific style programmatically.

03. Use a provider

There are three types of UI providers in Tailwind NG : component provider, module provider and application wide provider.

All providers have IntelliSense for all available properties and possible values so you won't need to remember them.

Use a provider when you need to apply a custom style to all instances of the component in a scope such as your host component, module or application wide. This is the most generic and recommended way.
