Customizations Guide

Tailwind NG provides a wide range of predesigned components to help you get started quickly. However, you may need to customize a component to match your design system or brand guidelines.

In this guide, you will learn the various ways you can customize Tailwind NG components to create a unique design for your application, make it stand out and fit your project's specifics needs.

These approaches are listed by order of precedence and apply to any component you may need to customize. Depending on your need, choose the best approach or combine them to get the desired outcome.

This guide assumes you have completed the getting started guide and already installed a theme in your project.

01. Using class attribute

Use the class attribute when you need to customize a specific component using utility classes.

Tailwind NG provide a built-in and well-tested mechanism that make merging all your custom classes easier, faster and accurate, while being very tiny (1.5 KB) compared to tailwind-merge (820 kB).

You can also set this property programmatically before the component initialization.

02. Using CSS selectors

Use any valid css selectors to set a rule that will customize multiple instances of a component at once.

You can also set these css rules in a stylesheet.

03. Using the component provider

Use the component provider when you need to achieve fine-grained customizations through dependency injection. Tailwind NG provide two kind of providers that let you apply customizations at any level of your application.
  1. application UI provider, exported as provideUI .
  2. component provider, exported in two flavours as follows:
    • provideComponentName . 'Standalone' provider, ideal for customizing outside the provideUI function.
    • withComponentName . 'Feature-like' provider, ideal for customizing inside the provideUI function.
